# Support Zigflow

Zigflow is an independently maintained open-source project. It is not a
Temporal product and receives no institutional funding.

Keeping Zigflow correct, compatible and well documented requires sustained
effort: tracking Temporal SDK changes, maintaining validation logic, writing
tests and keeping documentation accurate as the DSL evolves.

---

## Why Support Matters

Open-source infrastructure tooling incurs ongoing costs that are not visible
in day-to-day use:

- Temporal API compatibility must be verified against each SDK release
- Validation logic must stay in step with DSL specification changes
- Documentation must reflect actual behaviour, not aspirational behaviour
- Issues and pull requests require review and maintenance

Projects that depend on Zigflow in production benefit directly from this work.
Financial support makes long-term maintenance sustainable.

---

## Financial Support

The primary mechanism for financial support is
[GitHub Sponsors](https://github.com/sponsors/mrsimonemms).

Sponsorship funds:

- Ongoing maintenance and compatibility work
- Documentation and example improvements
- Security and dependency updates

There are no sponsorship tiers with feature commitments. Sponsorship supports
maintenance, not a roadmap.

---

## Corporate Support

Organisations using Zigflow in production are welcome to reach out via the
[GitHub repository](https://github.com/zigflow/zigflow) to discuss
consulting arrangements or dedicated support agreements.

No formal commercial offering currently exists. Enquiries will be considered
on a case-by-case basis.

---

## Non-financial Contributions

If financial contribution is not appropriate, the following are equally
valuable:

- **Bug reports**: detailed, reproducible reports in the
  [issue tracker](https://github.com/zigflow/zigflow/issues)
- **Pull requests**: fixes, documentation improvements and example additions
- **GitHub stars**: starring the
  [repository](https://github.com/zigflow/zigflow) increases discoverability
  and signals active use to the broader community
- **Visibility**: sharing Zigflow with teams who may benefit
- **Feedback**: opening
  [GitHub Discussions](https://github.com/zigflow/zigflow/discussions)
  to report real-world usage patterns

All contributions are made under the
[Apache 2.0 licence](https://github.com/zigflow/zigflow/blob/main/LICENSE).

## Contact

### Technical issues and feature requests

Open a [GitHub issue](https://github.com/zigflow/zigflow/issues). This is
the primary channel for bug reports, feature requests and technical questions.
Direct messages and email are not monitored for technical support.

### Community discussion

The [Zigflow Slack community](https://slack.zigflow.dev) is available for
informal discussion. It is a community space and is not an official support
channel.
